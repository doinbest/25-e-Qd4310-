# MaixCAM2–STM32 串口协议 V1

> 状态：已确认的联调协议方案，尚未执行到代码。  
> 最后确认：2026-07-19。  
> 本文替代项目中历史的 MaixCAM 串口帧约定；实施时不保留旧协议兼容层。

## 1. 设计边界

- MaixCAM2 是感知节点，只发送视觉观测和云台 IMU 数据；不直接发送电机速度、电流或 PWM。
- STM32 是模式、PID、电机输出与安全策略的唯一主控。
- 激光实际由相机端 B3 引脚驱动，但必须由 STM32 的控制帧授权。
- 本版本只约定通信格式、收发频率和数据含义；**不在本阶段实现或修改 PID、云台电机控制、超时停机与激光超时策略**。
- 视觉 `error_x/error_y` 的坐标正方向与其映射到 yaw/pitch 电机正方向的关系，尚未上板确认；实施时必须以实物测试结果明确记录，不能凭经验取反。

## 2. 物理连接与链路参数

| 项目 | 约定 |
| --- | --- |
| MaixCAM2 发往 STM32 | UART4 TX（A21）→ STM32 USART3 RX（PB11） |
| STM32 发往 MaixCAM2 | STM32 USART3 TX（PB10）→ UART4 RX（A22） |
| 地线 | 两端必须共地 |
| 波特率 | `115200 bps` |
| 串口格式 | `8N1`，无流控 |
| 通信方式 | 全双工，二进制帧 |

暂不升级至 921600 bps。以本协议最大发送频率估算，115200 bps 有充足余量；先完成稳定性测试，再决定是否更改。

## 3. 统一帧格式

所有多字节整数使用 **大端序（MSB first）**。不使用浮点数，避免端序、精度和 NaN 等跨端歧义。

```text
Byte 0..1    SOF       0x55 0xAA
Byte 2       VER       协议版本，V1 固定为 0x01
Byte 3       TYPE      帧类型
Byte 4       FLAGS     类型相关状态位
Byte 5       SEQ       本方向递增的 8 位帧序号，自然回绕
Byte 6       LEN       PAYLOAD 字节数
Byte 7..N    PAYLOAD   变长负载
Byte N+1..2  CRC16     CRC-16/CCITT-FALSE，高字节在前
```

### CRC 定义

| 项目 | 值 |
| --- | --- |
| 算法 | CRC-16/CCITT-FALSE |
| 多项式 | `0x1021` |
| 初始值 | `0xFFFF` |
| RefIn / RefOut | `false / false` |
| XorOut | `0x0000` |
| 校验覆盖范围 | 从 `VER`（Byte 2）至 `PAYLOAD` 最后一个字节；不含 SOF 和 CRC 本身 |
| CRC 字节序 | 高字节在前 |

接收方仅在帧头、版本、长度和 CRC 全部通过后，才提交该帧的数据与接收时间戳。校验失败、未知类型或长度不匹配的帧直接丢弃并继续搜索下一组 `0x55 0xAA`。

## 4. 帧类型

### 4.1 MaixCAM2 → STM32：视觉观测（`TYPE = 0x01`）

此帧在每次完成视觉处理后发送；没有识别到目标时也发送，用 `valid=0` 明确告诉 STM32 当前观测无效。频率不超过 100 Hz。

| 字段 | 字节数 | 类型 | 说明 |
| --- | ---: | --- | --- |
| `FLAGS.bit0` | 0 | `bool` | `target_valid`：本帧目标是否有效 |
| `FLAGS.bit1` | 0 | `bool` | `mapping_valid`：靶面映射/单应性结果是否有效 |
| `FLAGS.bit2` | 0 | `bool` | `target_locked`：目标筛选器是否已锁定当前靶面 |
| `FLAGS.bit3` | 0 | `bool` | `control_fresh`：相机已收到新的 STM32 控制帧 |
| `error_x` | 2 | `int16` | 图像/标定平面中横向误差，单位与正方向须在实现时固化 |
| `error_y` | 2 | `int16` | 图像/标定平面中纵向误差，单位与正方向须在实现时固化 |
| `applied_control_seq` | 1 | `uint8` | 相机当前实际采用的最近一帧 STM32 控制帧 `SEQ` |

`LEN = 5`，总帧长为 `14` 字节。

### 4.2 MaixCAM2 → STM32：云台 IMU（`TYPE = 0x02`）

云台 IMU 刚性安装在 pitch 组件上，与相机和激光器同一运动组件。此帧频率为 200 Hz。

| 字段 | 字节数 | 类型 | 说明 |
| --- | ---: | --- | --- |
| `FLAGS.bit0` | 0 | `bool` | `imu_valid`：本帧陀螺仪数据有效 |
| `FLAGS.bit1` | 0 | `bool` | `imu_calibrated`：IMU 已完成所需校准 |
| `gyro_x_dps_x10` | 2 | `int16` | X 轴角速度，单位 `0.1 dps` |
| `gyro_y_dps_x10` | 2 | `int16` | Y 轴角速度，单位 `0.1 dps` |
| `gyro_z_dps_x10` | 2 | `int16` | Z 轴角速度，单位 `0.1 dps` |

`LEN = 6`，总帧长为 `15` 字节。

IMU 轴与 STM32 的 yaw/pitch 轴映射和符号目前尚未确认。实施前必须通过单轴转动试验记录对应关系。

### 4.3 STM32 → MaixCAM2：控制与激光授权（`TYPE = 0x81`）

STM32 周期发送，频率为 50 Hz。该帧只表达模式和激光授权，不在相机侧产生电机控制量。

| 字段 | 字节数 | 类型 | 说明 |
| --- | ---: | --- | --- |
| `FLAGS.bit0` | 0 | `bool` | `laser_authorized`：允许相机端 B3 输出激光 |
| `mode` | 1 | `uint8` | 当前工作模式 |
| `circle_progress` | 2 | `uint16` | 圆周运动进度；仅在圆周模式下有效，定义为 `0..65535` 对应一整周 |

`LEN = 3`，总帧长为 `12` 字节。

模式枚举：

| 值 | 名称 | 含义 |
| ---: | --- | --- |
| `0x00` | `MODE_IDLE` | 待机 |
| `0x01` | `MODE_CENTER` | 靶心瞄准 |
| `0x02` | `MODE_CIRCLE` | 圆周同步 |

## 5. 解析与并发约定

- UART 为全双工链路；两端分别维护独立的发送序号和接收解析状态。
- MaixCAM2 只保留一个串口接收入口；发送需串行化，防止多处代码写入同一 UART。
- STM32 使用 DMA 环形接收缓冲区；中断仅搬运字节、推进写入位置或置位，完整帧解析放在主循环/周期任务中。
- 不在中断或 UART 回调中直接改变 PID、云台输出或工作模式；先完成合法帧提交，再由主控制周期读取最新已提交数据。
- `SEQ` 用于诊断重复、跳帧和命令是否真正被相机应用；它不是重传机制。
- 视觉帧中的 `applied_control_seq` 用于确认相机已应用的最新控制帧，便于定位“控制命令已发出但接收端仍停留旧状态”的问题。
- 单帧解析不能依赖帧尾；通过 `LEN + CRC` 完成定界与校验。收到异常字节流时应尽快重新搜索 SOF，不能阻塞等待。

## 6. 初始频率与带宽

| 数据方向 | 类型 | 频率上限 | 单帧长度 | 估算有效载荷速率 |
| --- | --- | ---: | ---: | ---: |
| 相机 → STM32 | 视觉 | 100 Hz | 14 B | 1400 B/s |
| 相机 → STM32 | IMU | 200 Hz | 15 B | 3000 B/s |
| STM32 → 相机 | 控制 | 50 Hz | 12 B | 600 B/s |

按 UART 8N1 的 10 bit/byte 估算，相机到 STM32 约占 `38%` 链路时间，STM32 到相机约占 `5%`。因此 V1 维持 115200 bps。

## 7. 实施与联调验收

实施时需要同时替换相机端与 STM32 端的历史协议，不能保留混用：

1. 为 CRC-16、整数大端编解码和三个帧类型准备固定测试向量。
2. 单元测试或串口模拟覆盖：正常帧、半帧、粘包、错误 CRC、错误长度、垃圾字节、SOF 跨缓冲区、重复序号与序号回绕。
3. 验证 STM32 控制帧的 `SEQ` 能被相机在后续视觉帧的 `applied_control_seq` 中回显。
4. 在 115200 bps 下按 100 Hz 视觉、200 Hz IMU、50 Hz 控制双向连续运行至少 30 分钟，记录 CRC 错误、丢帧、解析重同步次数和序号跳变。
5. 上板确认视觉 `error_x/error_y` 与 yaw/pitch、电机和 IMU 三轴的正方向，再开始 PID 整定。

## 8. 待后续任务确定的安全策略

以下内容刻意不在协议 V1 定稿时实现，待 PID 调参和正式控制任务开始时一并指定：

- 视觉帧、IMU 帧和控制授权帧各自的超时阈值。
- 数据超时、识别丢失、CRC 连续失败后的电机/PID 行为。
- 激光授权失效时 B3 的默认状态与延迟。
- 进入/退出 `MODE_CENTER`、`MODE_CIRCLE` 时的积分清零、速度限幅和缓停策略。

无论采用何种细则，后续安全策略必须由 STM32 统一决策，且不得继续使用过期视觉数据。
