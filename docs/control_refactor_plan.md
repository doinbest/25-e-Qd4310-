# 控制代码重整计划

> 状态：待实施。本文记录当前已确认的方案，后续可直接修改。

## 总体结构

- 拆分现有超长 `control.c`：
  - `line_control.c/.h`：底盘巡线、编码器距离、转弯状态机、巡线 PID 调用。
  - `gimbal_control.c/.h`：MaixCAM2 数据有效性、双轴级联 PID、QD4310 速度指令、安全状态与云台调试。
  - `control.c/.h`：仅保留兼容入口与公共复位接口，删除重复实现和全部失效的 `#if 0` 云台方案。
- 保留现有 `bsp_pid` 作为基础 PID 库；把所有可现场调节参数集中到两个控制模块顶部，标注单位、方向和调节影响。
- 维持裸机主循环：巡线传感器更新、巡线任务、云台任务均在主循环或定时调度中执行；中断只采集数据和更新时间戳。

## 底盘巡线

- 保留现有状态机并整理为：`READY → FOLLOW → APPROACH → STOP_FOR_TURN → TURN → REACQUIRE → DONE/LOST`。
- 角点条件改为严格判断灰度第 1–4 路均为黑线，连续确认后才进入补偿段；保留角点重触发距离，避免同一横线重复计数。
- 继续使用编码器平均距离，转向阶段从总里程中扣除原地旋转造成的伪里程；每 4 次转弯计为 1 圈，按目标圈数完成后以最终直线距离回到起点并停车。
- 保留左右轮速度闭环、灰度 PD 巡线、陀螺仪 90° 转角闭环、输出限幅、零速防反接和丢线停车。
- 将当前已实测参数原样迁移；`TURN_COMPENSATION_MM` 单独集中，当前值不在重构时擅自改为 150 mm。上板后以 150 mm 为理论参考重新标定。
- 巡线校准、圈数、启停和速度档保留为可由上位机/API 调用的接口，但不再占用实体按键。

## 云台与通信

云台固定采用一套双轴、速度模式控制链路：

```text
MaixCAM2 像素误差
→ STM32 像素外环（目标云台角速度，deg/s）
→ STM32 IMU 角速度内环（目标电机速度，rpm）
→ QD4310 SPEED 指令 0x04
→ QD4310 内置速度环与电流环
```

- 删除 STM32 闭环中所有 `QD4310_SetCurrent()`、`SetLowSpeed()` 与“X/Y 轴使用不同执行模式”的旧逻辑；电流模式不保留为正常控制或调试入口。
- MaixCAM2 统一发送带校验的误差帧：`FF FF 10 FLAGS SEQ ERR_X_H ERR_X_L ERR_Y_H ERR_Y_L CRC EE`。
  - `ERR_X/ERR_Y` 为大端 `int16_t`，单位 px；正值分别表示目标在瞄准参考点右侧或下侧。
  - `FLAGS.bit0` 为目标有效；`SEQ` 仅用于识别新帧；`CRC` 为从 `TYPE` 到最后一个数据字节的逐字节异或。
  - STM32 只接收已完成视觉补偿后的误差，不再重复使用“靶心减激光点”或重复叠加安装偏移。
- 重写 `maixcam.c/.h` 接收状态机，使视觉帧、IMU 角速度帧和姿态帧各自具有固定长度、CRC、序号、有效标志与最新时间戳。
- 像素外环只在收到新 `SEQ` 时更新，目标为零误差；视觉帧过期不继续积分旧误差。
- IMU 内环固定 200 Hz，直接使用 MaixCAM2 转发的陀螺仪角速度；X/Y 对应的 IMU 轴、像素符号、IMU 符号和电机符号全部集中为宏，先按当前映射迁移，最终用低速单轴测试确认。
- QD4310 速度命令固定 50 Hz：内环每 5 ms 更新最新速度目标，但每 20 ms 仅发送一次 `QD4310_SetSpeed()`，避免超过手册建议的控制频率。
- 视觉超时、IMU 超时、连续 UART 发送失败或模式切换时：清空两级 PID，先发两轴 `0 rpm`，20 ms 后失能；不允许继续沿用旧目标。
- UART 资源固定为：USART2 为 X 轴 QD4310，UART5 为 Y 轴 QD4310，USART3 为 MaixCAM2。移除灰度模块对 UART5 的发送、`printf` 和串口所有权；灰度采样仅保留实际无 MCU 传感器驱动。

## 按键与调试

- 按云台优先恢复为：
  - K1：调试页循环（Run、X 角速度、Y 角速度、X 像素、Y 像素）。
  - K2：X 轴使能/失能。
  - K3：Y 轴使能/失能。
  - K4：CENTER/CIRCLE 任务切换，切换前停止闭环。
  - K5：当前页的启动/停止。
- 单轴调参严格按“IMU 速度内环 → 像素外环 → 双轴 CENTER → CIRCLE”顺序；调试页显示本环目标、实际、输出、通信新鲜度和故障状态。
- 巡线参数、灰度校准和竞赛复位改由上位机调试命令或明确 API 调用，不与云台按键冲突。

## 验证

- 静态检查：确保仅有一套云台实现、无重复公共符号、无失效 `#if 0` 控制代码，新增源文件加入 Keil Target，并确认 `Hardware/qdrive/QD4310.c` 已参与链接。
- 巡线测试：直线、四路角点确认、补偿距离、90° 转弯、重新捕线、丢线、1–5 圈完成、急停和校准数据更新。
- 通信测试：正确帧、CRC 错帧、截断帧、重复序号、视觉超时、IMU 超时和 UART 发送失败均不得驱动电机。
- 云台上板测试：先限速、空载、机械限位受控状态下分别验证两轴的 IMU 正负反馈，再调内环 P/I，随后调像素环，最后验证 50 Hz 指令节拍、失效置零再失能和双轴瞄准。

## 已锁定的默认项

- 云台正式执行模式：QD4310 `SPEED`，不自动切换低速或电流模式。
- 速度指令频率：50 Hz；IMU 内环：200 Hz。
- 通信输入：带 CRC/序号的有符号像素误差帧。
- 失效策略：先零速度，后失能。
- 按键归属：云台优先。
