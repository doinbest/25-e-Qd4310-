# 25E 二维无刷云台双环联调使用说明

## 1. 当前控制结构

```text
MaixCAM2 靶框识别与视差补偿
        │ CENTER_X/Y 或 AIM_X/Y
        ▼
STM32 像素外环：图像中心 - 瞄准点像素坐标
        │ 目标角速度 deg/s
        ▼
STM32 角速度内环：目标角速度 - MaixCAM2 陀螺仪角速度
        │ 目标电流 A
        ▼
QD4310 内置电流环 → X/Y 两轴电磁力矩
```

X、Y 两轴现在都使用 QD4310 电流模式。QD4310 内置电流环不改，STM32 新增的是外部角速度环。像素 PID 与角速度 PID 共用一套轴控制函数，只有 PID 实例、IMU 轴映射、符号和电流限幅不同。

基础瞄准任务使用 `CENTER_X/Y`；画 6 cm 半径圆使用 `AIM_X/Y`。圆周相位由 STM32 编码器里程产生，按一圈 `4000 mm` 映射到 `0~10000`。

## 2. 三种频率不要混淆

| 项目 | 当前值 | 含义 |
|---|---:|---|
| IMU 芯片内部 ODR | 416 Hz | IMU 寄存器中新数据的更新频率 |
| Maix 应用读取/滤波 | 200 Hz | Python 每 5 ms 读取一次并更新 12 Hz 低通 |
| Maix IMU 串口帧 | 200 Hz | 每 5 ms 发送一帧 13 字节 IMU 数据 |
| STM32 角速度内环 | 100 Hz | 每 10 ms 用最新有效 IMU 数据计算一次电流 |
| STM32 控制帧 | 100 Hz | 每 10 ms 向 Maix 发送当前模式和圆周进度 |
| 视觉坐标帧 | 视觉 FPS | 每完成一次视觉处理发送一帧 |

屏幕 `TX:xxxHz` 显示 Maix 线程实测的平均 IMU 发送频率。目标值是 200 Hz，但 Python 调度、I2C 和 UART 阻塞会造成小幅波动。

STM32 与 Maix 不做“先 IMU、后控制”的硬同步。STM32 每 10 ms 读取接收中断保存的最近样本，并检查时间戳；200 Hz 发送意味着正常情况下每轮控制前会到达 1～2 帧新 IMU 数据。IMU 超过 30 ms 未更新时立即发零电流，视觉超过 200 ms 未更新时停止闭环。

## 3. UART 接线

| MaixCAM2 | STM32F103RCT6 | 说明 |
|---|---|---|
| A21 / UART4_TX | PB11 / USART3_RX | Maix 发送视觉与 IMU |
| A22 / UART4_RX | PB10 / USART3_TX | STM32 发送模式与进度 |
| GND | GND | 必须共地 |

两端参数均为 `115200, 8N1`。UART4 继续留给 JY901，不与 MaixCAM2 共用。

## 4. 串口协议

所有多字节数均为高字节在前。CRC 是从 `TYPE` 到最后一个数据字节逐字节异或；帧头和帧尾不参与。

### 4.1 视觉帧，Maix → STM32，15 字节

```text
FF FF 10 FLAGS SEQ CENTER_XH CENTER_XL CENTER_YH CENTER_YL
         AIM_XH AIM_XL AIM_YH AIM_YL CRC EE
```

- `CENTER_X/Y`：靶心模式的视差补偿后相机瞄准点。
- `AIM_X/Y`：画圆模式的视差补偿后相机瞄准点。
- `SEQ`：视觉帧独立序号。
- STM32 要求目标、映射、瞄准点和控制链路标志有效，并检查当前模式标志。

### 4.2 IMU 帧，Maix → STM32，13 字节

```text
FF FF 11 IMU_FLAGS SEQ GX_H GX_L GY_H GY_L GZ_H GZ_L CRC EE
```

- `GX/GY/GZ`：有符号 `int16`，单位 `0.01 deg/s`。
- `IMU_FLAGS bit0`：本次读取有效；无效帧不会刷新 STM32 安全时间戳。
- `IMU_FLAGS bit1`：已加载陀螺仪零偏校准。
- `SEQ`：IMU 独立序号，与视觉序号无关。

### 4.3 控制帧，STM32 → Maix，9 字节

```text
FF FF 21 MODE PROGRESS_H PROGRESS_L SEQ CRC EE
```

- `MODE=0`：IDLE，关闭激光。
- `MODE=1`：CENTER，靶心瞄准。
- `MODE=2`：CIRCLE，按 `PROGRESS=0~10000` 计算圆周点。
- Maix 250 ms 收不到有效 `0x21` 帧会回到 IDLE。

视觉帧和 IMU 帧不会相互冲突：二者类型和固定长度不同，Maix 发送时还有互斥锁，STM32 按帧头、类型、固定长度、CRC、帧尾解析。

115200 下一个字节按 10 bit 计算。200 Hz IMU 占 `13×10×200=26000 bit/s`；若视觉为 30 FPS，再占 `15×10×30=4500 bit/s`，Maix 发送方向合计约占 26.5%。STM32 返回控制帧占 `9×10×100=9000 bit/s`，全双工 UART 余量足够。

## 5. 像素目标与实际值

当前视觉代码不识别画面中的真实激光光斑。它检测靶框，通过单应变换和固定的镜头—激光安装偏移，计算“相机应该对准哪个像素，实际激光才会落到目标上”。

因此像素 PID 必须这样接：

| 模式 | PID 目标值 | PID 实际值 |
|---|---:|---|
| X 像素环 | 160 px | `CENTER_X` 或 `AIM_X` |
| Y 像素环 | 120 px | `CENTER_Y` 或 `AIM_Y` |
| X 角速度环 | X 目标角速度 | 映射后的 IMU X 云台轴角速度 |
| Y 角速度环 | Y 目标角速度 | 映射后的 IMU Y 云台轴角速度 |

PID 库内部误差统一为 `target - actual`。不要再计算“靶心坐标减激光点坐标”，也不要在 STM32 重复叠加 16.5 mm 视差补偿。

激光需要在正式任务中打开以投射到靶面，但不是为了让当前视觉程序识别激光点。Maix 仅在收到有效 CENTER/CIRCLE 模式且靶框有效时自动打开 B3 PWM。

## 6. 五个按键

| 按键 | 功能 |
|---|---|
| K1 | 调参页面循环：Run → X角速度 → Y角速度 → X像素 → Y像素 |
| K2 | X 轴电机使能/失能 |
| K3 | Y 轴电机使能/失能 |
| K4 | 任务选择：CENTER ↔ CIRCLE；切换时先停止当前闭环 |
| K5 | 启动/停止 K1 当前页面对应的闭环 |

原有小车速度档位、灰度校准和小车启停按键逻辑已从这五个键移除。Run 页面 K5 启动双轴；单轴调参页面 K5 只使能并运行所选轴。

## 7. 野火上位机调参

上位机“切换周期/模式”命令和 K1 使用相同顺序：

1. X 角速度环
2. Y 角速度环
3. X 像素环
4. Y 像素环
5. 回到 Run

建议严格按内环到外环调节：

1. 拆下高风险负载或做好机械限位，将电流限幅保持在 `0.30 A`。
2. 进入 X 角速度页，先给 `+5 deg/s`，再给 `-5 deg/s`。确认目标为正时反馈方向也为正；不对就修改 IMU 轴或符号宏。
3. X 内环先只加 P，让实际角速度快速跟随但不持续振荡；再加很小 I 消除负载静差，D 最后按需增加。
4. 同样方法调 Y 角速度环。
5. 进入 X 像素页。目标固定 160，实际是相机返回 X 瞄准点；已调好的 X 角速度内环继续工作。
6. 进入 Y 像素页，目标固定 120。
7. 最后回 Run，先 CENTER，再 CIRCLE。

默认初值位于 `Hardware/PID/bsp_pid.c`：像素环 `Kp=0.30`，角速度环 `Kp=0.010, Ki=0.0001`。这些只是低电流起步值，不是带负载最终参数。

方向与限幅位于 `Hardware/control.c`：

```c
#define VISION_X_GYRO_AXIS           MAIXCAM_GYRO_AXIS_Z
#define VISION_Y_GYRO_AXIS           MAIXCAM_GYRO_AXIS_X
#define VISION_X_GYRO_SIGN           1.0f
#define VISION_Y_GYRO_SIGN           1.0f
#define VISION_X_PIXEL_TO_RATE_SIGN -1.0f
#define VISION_Y_PIXEL_TO_RATE_SIGN  1.0f
#define VISION_X_CURRENT_LIMIT_A      0.30f
#define VISION_Y_CURRENT_LIMIT_A      0.30f
```

如果只手动转动某一机械轴，先观察 OLED/上位机是哪一个 `GX/GY/GZ` 明显变化，再修改 `VISION_*_GYRO_AXIS`。若目标角速度为正而电机加速后反馈变负，修改该轴 `GYRO_SIGN`；若闭合像素环后目标越跑越远，修改对应 `PIXEL_TO_RATE_SIGN`。

## 8. OLED 页面

Run 页面显示：当前 CENTER/CIRCLE、控制状态、有效标志、X/Y 像素实际值、yaw/pitch 角速度、两轴电流、两轴使能和视觉/IMU 帧计数。

四个调参页面分别显示本环的目标、实际、输出电流和 P/I/D。野火曲线也严格发送本环的目标与实际，不再发送像素误差冒充“实际值”。

## 9. 安全检查与失效行为

- 首次联调先不装大负载，确保机械限位和急停手段有效。
- IMU 首次使用时将 `IMU_CALIBRATE_ON_BOOT=True`，静置约 10 秒完成保存，然后改回 `False`。
- 视觉丢失、模式不匹配、CRC 错误或 IMU 超时都会阻止闭环；IMU 超时立即发送零电流。
- 两轴电流输出有绝对限幅和每 10 ms `0.01 A` 的斜率限制。
- 调速度内环时激光保持关闭；进入 CENTER/CIRCLE 且视觉有效后才打开。
- QD4310 驱动每次命令会等待反馈，若电机不返回 10 字节反馈，单轴可能阻塞到 5 ms；双轴 100 Hz 联调时必须确认反馈及时。

## 10. Keil 工程必须补的一步

当前 `QD4310.c` 文件已存在并可单独通过 ARMCC 编译，但 `MDK-ARM/21F.uvprojx` 尚未把它加入 Target，所以整工程链接会报告：

```text
Undefined symbol QD4310_Enable
Undefined symbol QD4310_Disable
Undefined symbol QD4310_SetCurrent
```

在 Keil 左侧工程树中选择合适的 Hardware 组，执行 **Add Existing Files to Group**，加入：

```text
Hardware/qdrive/QD4310.c
```

不要只加入头文件。加入后重新 Build；本次修改涉及的 `protocol.c`、`bsp_debug.c`、`bsp_key.c`、`main.c`、`maixcam.c` 和 `control.c` 已通过编译阶段。

## 11. 推荐上电顺序

1. MaixCAM2 与云台静置，确认校准已加载，画面 `TX` 接近 200 Hz。
2. STM32 上电，Run 页确认 IMU 帧计数持续增加，静止角速度接近 0。
3. 不启动像素环，依次完成 X、Y 角速度单轴正负目标测试。
4. 完成 X、Y 像素环调参。
5. 回 Run，K4 选 CENTER，K5 启动双轴。
6. CENTER 稳定后再切 CIRCLE，并确认 `0~10000` 进度对应一整圈。
